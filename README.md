# Homelab Stacks

Коллекция качественно написанных Docker Compose стеков для моего домашнего сервера. Проект использует Ansible для централизации всех секретов и переменных окружения в одном файле, а также для идемпотентного развертывания стеков.

## Архитектура

1. **Ansible (`ansible/`)**: Содержит плейбуки, таски и переменные. Отвечает за:
   - Развертывание стеков на сервере.
   - Рендер переменных окружения из шаблонов `.env.j2`.
   - Идемпотентное создание необходимых директорий с правильными правами доступа.
   - Обнаружение изменений в конфигурационных файлах, переменных окружения, секретах и `docker-compose.yaml` (Ansible сам определяет, требуется ли пересоздать, перезапустить или пропустить сервисы. Подробнее в [ansible/README.md](ansible/README.md)).

2. **Стеки (`stacks/`)**: Наборы сервисов, описанные в `docker-compose.yaml`.

## Использование

### Требования

* Сервер на Linux (протестировано на Debian 13).
* Установленные пакеты: Docker и Docker Compose из официального репозитория [Docker](https://docs.docker.com/engine/install/), а также Ansible.
* Проект рассчитан на работу от имени обычного пользователя (не root) с UID/GID 1000:1000. Все команды в репозитории и конфигурации прав доступа ориентированы на эти значения. У пользователя должны быть права на использование `sudo`.

### Конфигурация

Скопируйте пример файла с переменными и установите свои значения. Больше информации можно найти в README каждого отдельного стека:
```bash
cp ansible/vars.example.yaml ansible/vars.yaml
```

### Развертывание

```bash
cd ansible
sudo ansible-playbook playbook.yaml # Развернет все стеки
sudo ansible-playbook --tags "reverse-proxy" # Развернет только стек reverse-proxy (замените на название нужного вам стека)
```

### Обновление

В случае изменения конфигурационных файлов, переменных окружения, секретов или самого `docker-compose.yaml`, просто повторите команды из этапа развертывания. Ansible автоматически определит, что именно требуется пересоздать, перезапустить или пропустить.

### Структура данных

Постоянные данные контейнеров по умолчанию монтируются в директорию `/mnt/docker-volumes`. Необходимые директории с заданными правами доступа создаются автоматически с помощью Ansible.
